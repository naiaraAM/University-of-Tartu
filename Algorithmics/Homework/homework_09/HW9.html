By parameter I guess it refers to a numeric value and not a visual representation. In my opinion, there are 3 parameters that describe a graph, that are the conectivity boolean (true/false), number of nodes and average degree.

Add some characterization model - which? What does it add?¶

I added the "closeness centrality" which measures how close a node is to all other nodes in the graph, based on the shortest path. A node with high closeness centrality can reach all other nodes faster than others. It helps identify nodes that can quickly spread information across the network.

Ex 4¶

Greedy analysis¶

A greedy algorithm can be defined as a really optimistig algorithm, due to its assumption that the next optimal step will lead to the optimal solution. It does not consider the whole overview of a problem and it is easy to make it fail. Besides, once the decision is made, it cannot be revesersed.

On easy example is going through a maze. A greedy algorithm can go any direction, but once a step is made i cannot be undo. If the step taken leads to no solution, it will stop and say that there is no solution, even if it was possible to solve the problem. It is true that greedy algorithms FIND an optimal solution but not always.

I create a simple example of searching the most weighted path from a node. The graph is a tree. I used for the optimal solution DFS. I represented both outcomes in a GIF (sorry for the infinite loop). The number that is not really well seen is 1000.

In the case of the greedy algorithm is easy to make it fail, by puting the weigth of one of the child nodes big enough so it selects it (in my case big enough is one unit bigger). Then if the children nodes of the non selected one are bigger by just 2 units, the algorithm will not find the optimal solution, as it is the case.
